ИНСТРУКЦИЯ ПО РАЗВЁРТЫВАНИЮ СИСТЕМЫ УЧЁТА ОБОРУДОВАНИЯ

1. Что входит в проект

Проект состоит из двух частей:
- серверная часть на Python (FastAPI) в папке `DB_Utills-master`;
- клиентская часть на React + TypeScript + Vite в папке `web_frontend`.

Для работы системы нужны:
- PostgreSQL;
- Python;
- Node.js;
- две запущенные части приложения: бэкенд и фронтенд.

2. Что нужно установить на компьютер

Перед началом убедитесь, что на ПК установлено:
- Python 3.11 или новее;
- Node.js 20 LTS или новее;
- PostgreSQL 14/15 или новее;
- любой удобный архиватор, если проект передан в архиве;
- браузер Google Chrome, Microsoft Edge или Firefox.

3. Подготовка базы данных PostgreSQL

По умолчанию проект ожидает следующую конфигурацию БД:
- хост: `127.0.0.1`
- порт: `5432`
- база данных: `equipment_accounting`
- пользователь: `postgres`
- пароль: `1234`

Если на компьютере преподавателя другие логин или пароль, их нужно заменить в файле `DB_Utills-master\.env`.

Как подготовить БД:
1. Установить PostgreSQL.
2. Создать базу данных с именем `equipment_accounting`.
3. Создать пользователя PostgreSQL или использовать уже существующего.
4. Выдать этому пользователю права на созданную базу.

Если используется стандартный пользователь `postgres`, то достаточно создать базу данных `equipment_accounting` и задать пароль, совпадающий с тем, что указан в `.env`.

4. Настройка файла окружения

Откройте файл `DB_Utills-master\.env` и проверьте значения:

```env
DB_HOST=127.0.0.1
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=1234
DB_NAME=equipment_accounting
FRONTEND_URL=http://localhost:5173
```

Если PostgreSQL установлен с другими параметрами, замените их на свои.

Важно:
- `FRONTEND_URL` должен указывать на адрес, где будет открыт фронтенд;
- для локального запуска обычно используется `http://localhost:5173`.

5. Запуск серверной части

Откройте PowerShell или терминал и перейдите в папку проекта:

```powershell
cd C:\EquipmentAccounting\DB_Utills-master
```

Далее выполните команды:

1. Создайте виртуальное окружение:

```powershell
python -m venv venv
```

2. Активируйте его:

```powershell
venv\Scripts\Activate.ps1
```

Если PowerShell запрещает запуск скриптов, можно выполнить команду:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

3. Установите зависимости:

```powershell
pip install -r requirements.txt
```

4. Запустите сервер:

```powershell
python main.py
```

После запуска сервер должен быть доступен по адресу:
- `http://localhost:8000`

Документация API:
- `http://localhost:8000/docs`

Что происходит при первом запуске:
- автоматически создаются таблицы в базе данных;
- создаются базовые записи, если их ещё нет;
- создаётся пользователь администратора по умолчанию.

Логин и пароль администратора по умолчанию:
- логин: `admin`
- пароль: `admin`

6. Запуск клиентской части

Откройте второе окно PowerShell и перейдите в папку фронтенда:

```powershell
cd C:\EquipmentAccounting\web_frontend
```

Установите зависимости:

```powershell
npm install
```

Запустите фронтенд:

```powershell
npm run dev
```

После запуска откройте в браузере:
- `http://localhost:5173`

7. Как проверить, что всё работает

После запуска двух частей системы проверьте:
- открывается главная страница фронтенда;
- выполняется вход под `admin/admin`;
- открывается список оборудования;
- работает переход к карточке оборудования;
- при необходимости открывается документация бэкенда по адресу `/docs`.


8. Частые проблемы и решение

Если фронтенд не открывается:
- проверьте, что `npm run dev` запущен;
- убедитесь, что порт `5173` свободен;
- откройте `http://localhost:5173`.

Если сервер не подключается к базе:
- проверьте, что PostgreSQL запущен;
- проверьте логин, пароль, имя базы и порт в `.env`;
- убедитесь, что база `equipment_accounting` реально создана.

Если не работает авторизация:
- проверьте, что запущен бэкенд на `http://localhost:8000`;
- убедитесь, что в браузере не блокируются cookie/localStorage;
- попробуйте удалить старые данные входа в браузере и войти снова.

Если PowerShell не даёт активировать виртуальное окружение:
- выполните `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`;
- затем снова запустите `venv\Scripts\Activate.ps1`.

10. Краткая последовательность установки

Если нужен совсем короткий порядок действий, то он такой:
1. Установить PostgreSQL, Python и Node.js.
2. Создать базу `equipment_accounting`.
3. Проверить файл `DB_Utills-master\.env`.
4. В папке `DB_Utills-master` создать и активировать виртуальное окружение.
5. Выполнить `pip install -r requirements.txt`.
6. Запустить бэкенд командой `python main.py`.
7. В папке `web_frontend` выполнить `npm install`.
8. Запустить фронтенд командой `npm run dev`.
9. Открыть `http://localhost:5173` и войти под `admin/admin`.

11. Примечание

Если проект запускается на другом компьютере, все адреса `localhost` остаются такими же, если и фронтенд, и бэкенд запускаются на этом же компьютере.
Если нужно открыть систему с другого устройства в локальной сети, тогда адреса в настройках надо будет заменить на IP-адрес компьютера, на котором запущен сервер.
